Why this exists

Generating an EVM-compatible keypair is a one-line call into any crypto library. Doing it at scale, safely, and with somewhere durable to put the results is a different problem — most scripts generate one wallet at a time with no concurrency model, and most tools leave you with a CSV you're scared to lose.

This is a single Go binary that does the unglamorous parts properly: a worker-pool engine that reuses memory instead of allocating per wallet, a vanity miner that can resume a search instead of restarting it, and a storage layer that starts at zero config and scales to Postgres without a rewrite.

What it does

engine

Concurrent generation

A worker-pool generator with sync.Pool reuse, built to sustain thousands of keypairs per second rather than thousands per minute.

mining

Resumable vanity search

Difficulty-aware probability math, OR-pattern matching, and search state that survives a restart instead of starting from zero.

derivation

HD wallets

Full BIP‑39 / BIP‑44 support — 12 or 24-word mnemonics, optional passphrase, standard derivation paths.

storage

SQLite, then Postgres

Zero-setup embedded storage by default. Switch on PostgreSQL when you need it, with connection-pool and database health monitoring built in.

export

Six export formats

Plaintext, CSV, JSON, and an encrypted Web3 Secret Storage keystore via go-ethereum's standard implementation.

interface

TUI and CLI, both first-class

A full interactive terminal UI for exploring it by hand, and non-interactive flags for scripting it into a pipeline.

Get running

Grab a prebuilt binary for Linux, macOS, or Windows from the releases page, or build it yourself with Go installed.

Latest release Full install guide 
# build from source
$ git clone https://github.com/vinayakkumar9000/EVM-Wallet-GENERATOR-BOT.git
$ cd EVM-Wallet-GENERATOR-BOT
$ go build ./...

# bulk generate, export to CSV
$ ./evmwalletbot -count 10000 -export csv

# resumable vanity search
$ ./evmwalletbot -vanity 0xCAFE -resume

Before you run this somewhere that matters

This tool generates real, valid private keys. Treat every output file and database as sensitive material — plaintext storage is an intentional design choice, not an oversight, and it's on you to encrypt at rest, keep machines offline, or switch to the encrypted keystore export when it matters.
